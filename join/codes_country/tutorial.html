Tutorials

All examples are included in the full distribution and the source distribution. Please refer to the download section.

Most of the examples are available for running from the command line as well as in a Web application. Please refer to the running section. Web applications are supposed to run on a servlet engine such as Tomcat installed on your local machine and listening on the port 8080. If it is located on a remote machine or listening to another port, please replace http://localhost:8080/ in the examples below by the relevant URL.

Check the prerequisites for running the tutorials in batch mode or within your servlet engine.

Feel free to change the examples with your own stuff ! It's the better way to understand and learn !

Checking tutorials prerequisites

Ensure that the libraries are reachable by the JVM as explained below.

Running Web applications

The Web applications in the tutorials are NOT standalone : ensure that the libraries expected are either placed on the shared directory of your servlet engine or copied on each web application.

Please download the full or the minimal distribution and refer to the install and running sections.

Building Web applications

However, if you have downloaded the full or the source distribution, you can also rebuild with Ant the standalone Web applications by launching the warallstandalone target of the build file : $REFLEX_HOME/build.xml. Please refer to the documentation of Ant.

Running Batch scripts

The Batch scripts in the tutorials are standalone : ensure that the libraries expected are in your CLASSPATH.

On Unix/Linux, the shell scripts are using the CLASSPATH set, if any ; if it is not the right one, just unset it : the launcher will build the expected CLASSPATH from the install directory.

On Windows, the REFLEX_HOME environment variable must be set.

Additional informations on running RefleX from the command line interface are available in the running section.
